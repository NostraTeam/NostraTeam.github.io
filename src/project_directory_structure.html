The following tree shows the directory structure of a Nostra project, and the paragraphs after the tree will explain the contents of each file and directory.

Note that some of the names in the directory tree are not always the same (e.g. because the file name contains the name of the project). In such a case, the name of the file in the tree will contain something like this: <project name>. The paragraph of the specific file will then contain further explanations what the file name actually will be.

Also, to enhance readability, in the following tree, regular file names are written in a regular font and directory names are written in a bold font. In addition to that, there are optional files. These are written in italics and their names are surrounded by round braces.

File Details

/

The root of the project. All files that are related to the project are in this directory of a subdirectory of it.

Note that / does not mean that the directory is the system's root directory - it is just a placeholder for the directory that the project files are in (it does not matter what that directory is called).

/cmake

A directory that contains files that are used by CMake. As of now, the files in this directory are exclusively files that will be configured by CMake using configure_file() .

/cmake/config.h.in

optional

A file that will be configured by CMake. The configured file can be included using
#include "nostra/<project name>/config.hpp
and it is meant to be used to inject variables from CMake into the C/C++ code.

To inject a CMake variable, a new macro needs to be created in this file:
#define MY_MACRO @<CMake variable name>@
where <CMake variable name> is the name of the variable that should be injected. Note that the name of the macro (MY_MACRO in this case) can be freely chosen - it is however common practice to make it an internal macro (thus making it start with <project prefix>_INTERNAL_) and then choose the rest of the name according to the CMake variable that will be injected.

Example: If a project has the prefix ntp and the injected variable is CMAKE_SOURCE_DIR, then a good name for the macro would be NTP_INTERNAL_SOURCE_DIR.

The default file that is generated by the CreateProject.cmake script creates a config.h.in that will inject the following variables:

CMake variable Macro name
PROJECT_VERSION_MAJOR <project prefix>_INTERNAL_VERSION_MAJOR
PROJECT_VERSION_MINOR <project prefix>_INTERNAL_VERSION_MINOR
PROJECT_VERSION_PATCH <project prefix>_INTERNAL_VERSION_PATCH
PROJECT_VERSION_TWEAK <project prefix>_INTERNAL_VERSION_TWEAK
