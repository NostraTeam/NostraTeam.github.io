Components

Components are a logical structure to organize features in a project.

A component contains all classes, functions and other code that belongs to a particular feature. Also, each component has a header file, a source file, (at least) one test and an example. The source file is always present, even if it would not have any code in it (for example if all functions are defined in the header because they are templates or marked as inline). In addition to that, each component has a name that must be appropriate for its content. Usually, each component has a "main" class. If so, that class would usually have the same name as the component.

Example:

A component that implements a linked list may have the name linked_list. In that case the header file and source file would be called linked_list.hpp and linked_list.cpp respectively. It would provide the class linked_list and perhaps iterator classes for the linked list. Additionally, there would (at least) be the test ntp.cpp.ex.s.linked_list.wt.1 (in this case, ntp was chosen as prefix) and the example linked_list.ex.
