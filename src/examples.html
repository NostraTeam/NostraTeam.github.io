Examples

Examples are meant to be an introduction for a user into a certain feature (or, in most cases, a component to be more precise).

Example Names

The name of an example is the name of the component that the example is for (or another appropriate name if the example is not for a single component) and a prefixed .ex (not to be confused with tests with the execution type ex). This prefix is used to identify an executable as an example.

Example Directory Structure

All examples are nested in the directory /examples/ (see here), but each example has its own directory structure that is nested in that directory. The directory structure for each example looks like this (bold names are directories). 
<example name>.d
└── <example name>.<language extension>

Unlike tests are examples meant to have only a single source file. This is meant to keep the example simple.
