README Structure

The readme file is a file that is meant to be read by a user (thus the name read-me) to give information about the library.

Structure

Each item in the following list is one chapter in the readme file. Also, after the list, there is a chapter on this page for each chapter in the readme file that describes it further.

  1. General information
  2. Requirements
    1. Usage Requirements
    2. Build Requirements
  3. How to build
  4. How to install
  5. Changelog
    1. Additions
    2. Removals
    3. Deprecations
    4. Improvements
    5. Fixes
    6. Known Issues

General information

This chapter provides, as the name implies, general information about the project. This information includes a brief description of the project and the website that the project is hosted on (e.g. the GitHub link if the repository is hosted on GitHub).

Note that, other than the other chapters, this chapter does not have its own chapter title in the readme file, it follows immediately after the file's headline.

Requirements

Usage Requirements
The additional requirements that are required to use the library (e.g. third party libraries or other Nostra libraries).
Build Requirements
The additional requirements that are required to build the library (e.g. Doxygen to build the documentation or a C++ compiler).

How to build

A description on how to build the entire project from source. This includes tests, documentation, examples and the library itself.

At this point, all of the CMake variables that are commonly used to modify the build should be listed as well.

Note that different operating systems may require different commands, not everybody is using the BASH or a similar shell.

How to install

A description on how to install the library when it was build from the description in the previous chapter and a description on how to install the library from an installer (if one is provided).

Changelog

The Changelog is a list of features that have been changed, added or removed from version to version. When a new version is released, the changelog of that version will be inserted at the top of the changelog.
Additions
A list of all features that have been added in the current version.
Removals
A list of all features that have been removed in the current version.
Deprecations
A list of all features that have been marked as deprecated in the current version.
Improvements
A list of all features that have been improved in the current version.
Fixes
A list of all bugs that have been fixed in the current version.
Known issues
A list of all known bugs that are still in the current version.

Markdown

Readme files are written in Markdown, a markup language that allows simple text formatting in plain text files. For Markdown's website see this link.
