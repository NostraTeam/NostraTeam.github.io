Naming Conventions

The following chapters describe the naming conventions for various naming classes (e.g. function or files).

Notations

There are a total of three notations used for names:
upper camel case
A name style where the single words within a name are written right next to each other and each new word starts with an uppercase letter. The remaining letters are written in lowercase. The first word also starts with an uppercase letter.

Example: ThisIsUpperCamelCase

lower camel case
Just like upper camel case, but the first letter of the entire name is a lowercase letter.

Example: thisIsLowerCamelCase

Lower camel case is currently not used for any of the names in the Nostra project.

underscore separated
A name style where the single words within a name are separated by underscores (the character "_"). Each word in the name is written in lowercase letters.

Example: this_is_underscore_separated

There is also a variant that only uses uppercase letters.

Example: THIS_IS_UNDERSCORE_SEPARATED

Prefixes and Suffixes

In addition to the name Notation, each name may also have a prefix and/or suffix. They do not follow the name notation, but instead are just added to the name exactly the way that they are given. The prefix and suffix is also optional, not every name has one.

The names that have the symbol ✓ in the column Has C prefix? have an additional prefix if the project is a C project. The prefix is the project prefix (in all lowercase or uppercase, depending on the style of the name notation) followed by an underscore. Note that these C prefixes are not part of the examples in the table, but a global function C would look like this ntp_my_function (if the project prefix is ntp).
